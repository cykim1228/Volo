# volo-app-dev — Volo 데스크톱 앱 구현

app-dev 에이전트가 UI를 구현할 때 따르는 방법론.

## 컨텍스트 확인 (먼저)
- `docs/ARCHITECTURE.md`(UI 스택·엔진 공개 API)와 `volo_engine/pipeline.py`(호출 시그니처)를 읽는다.
- 기존 `volo_app/`이 있으면 읽고 변경분만 수정.

## 핵심 원칙
- **엔진 분리 준수.** UI는 `volo_engine.pipeline.run(...)`을 호출하고 `progress_cb`를 구독할 뿐, STT/세그멘테이션을 재구현하지 않는다.
- **UI 비차단.** 무거운 추론은 워커 스레드(QThread)/별도 프로세스에서 실행하고, 진행률·상태 텍스트를 메인 스레드로 시그널 전달. UI가 "멈춘 듯" 보이지 않게 단계명("오디오 추출 중", "전사 중 42%")을 갱신.
- **경계면 정합성.** 엔진 반환 타입(`Subtitle` 등)을 그대로 소비. 필드 불일치 시 임의 변환 말고 architect/qa에 보고.

## 사용자 플로우 (MVP)
1. 영상 파일 드래그앤드롭 또는 선택(여러 개 → 배치).
2. 옵션: 모델 크기, 언어(자동/ko), 번역 대상(선택), 스타일 프리셋, 출력 포맷(SRT 기본).
3. "자막 생성" → 진행률 바 + 단계 상태.
4. 결과: cue 목록 미리보기(시작/끝/텍스트), 간단 인라인 편집(텍스트 수정).
5. 내보내기: 출력 폴더 선택 → SRT(및 선택 포맷) 저장 → "프리미어에서 임포트" 안내.

## 스택
- 기본: **PySide6**. 위젯: 드롭 영역(`QWidget`+dragEnter/drop), 옵션 폼, `QProgressBar`, cue 테이블(`QTableWidget`), 파일 다이얼로그.
- 진행률: `pipeline.run(..., progress_cb=cb)`의 콜백을 Qt 시그널로 브리지(워커 스레드 → UI).
- ARCHITECTURE.md가 다른 스택(Electron 등)으로 결정했으면 그 결정을 따른다.

## 에러 UX
- 엔진 예외(ffmpeg 없음/모델 실패)는 사용자 친화 다이얼로그로. 재시도/설치 안내 링크 제공.
